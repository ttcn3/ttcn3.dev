---
title: Why Go?
---

ntt's predecessor [K3](faq/what-is-k3) is mostly written in C++ (and some C). We experienced similar issues with C++ as [Google did](https://talks.golang.org/2012/splash.slide#11). We were looking for a programming environment, which:

* is fast
* is safe
* is easy to pick up
* is easy to maintain
* has a package manager
* comes with a build system
* comes with a test system
* has automatic memory management
* supports multiple platforms and operating systems
* has excellent support for concurrency
* has excellent support for networking

Go is a very good fit.
